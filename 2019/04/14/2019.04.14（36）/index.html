      今天还是继续学习了汇编语言的知识点，并结合reverse的题目和《深入理解计算机系统》的题目来学习，感觉对汇编语言的理解开始变得深入。今天主要学习的内容主要有：

  • x86-64下的通用寄存器与汇编指令
  • 条件码
  • 比较指令与测试指令

一、x86-64下的通用寄存器与汇编指令

1.x86-32与x86-64的数据类型宽度

2.x86-64的通用寄存器

  • 扩展了现有的，并增加了8个新的
  • %ebp/%rbp 不再是专用寄存器
    3.x86-32/64下的swap函数比较

不同点：

  • x86-32传参默认是通过内存来传参；
  • x86-64使用寄存器来传递参数，当参数少于7个时，参数从左到右放入寄存器：rdi、rsi、rdx、rcx、r8、r9。当参数为7个以上时，前面 6 个传送方式不变，但后面的依次“从右向左”放入栈中。x86-64 时无栈操作，被操作数据仍为32位，所以使用寄存器%eax、%edx以及movl指令

3.变形：x86-64下long int类型的swap

被操作位的数据是64位，所以使用寄存器%rax、%rdx，以及movq指令（“q”表示“4字”）

4.x86指令的特点

  • **支持多种类型的指令操作数：**立即数、寄存器、内存数据
  • 算术/逻辑指令可以以内存数据为操作数
  • 支持多种内存地址计算模式
    • Rb + S * Ri + D
    • 也可用于整数计算（如：leal指令）

二、条件码

  • CF（Carry  Flag）：进位位
  • SF（Sign    Flag）：符号位
  • ZF（Zero    Flag）：零位
  • OF（Overflow  Flag）：溢出位

☛ 这些条件码由算术指令隐含设置

addl    Src,Dest             addq    Src,Dest
类似的C语言表达式：
t = a + b （a = Src，b = Dest）
- CF 进位标志：可用于检测无符号整数运算的溢出 - ZF set if t == 0 - SF set if t  <  0 - OF set if：补码运算溢出（即带符号整数运算） （a > 0 && b >0 && t < 0）|| （a < 0 && b <0 && t> 0）

三、比较指令与测试指令

1.比较指令（Compare）指令

cmpl    Src2,Src1             cmpq    Src2,Src1
cmpl b,a 类似于计算a - b（但是不改变目的操作数） - 可用于无符号数的比较    ZF set if a == b    SF set if (a < b) < 0    OF set if two's complement overflow - (a > 0 && b < 0 && (a - b) < 0）|| （a < 0 && b > 0 && (a - b) > 0)

2.测试指令（Test）

testl    Src2,Src1             testq    Src2,Src1
计算Src1 & Src2并设置相应的条件码，但是不改变目的操作数 - ZF set when a&b == 0 - SF set when a&b < 0

test 指令使CF、OF为0
